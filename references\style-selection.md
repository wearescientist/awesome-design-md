# Style Selection Notes

Use this only when the project context does not clearly imply a style.

## Strong Defaults

- Technical dark console: `voltagent`
- Dense productivity dashboard: `linear.app`
- Command palette / launcher: `raycast`
- Code editor or AI coding surface: `cursor`
- Terminal / dev infra: `warp`
- Clean SaaS landing or docs: `vercel`
- Payments / B2B finance: `stripe`
- Crypto exchange / market UI: `coinbase` or `binance`
- Minimal AI lab product: `x.ai`
- Knowledge base / workspace: `notion`

## Avoid

- Do not pick luxury car styles for dense dashboards unless the user asks for a brand-heavy landing page.
- Do not pick consumer lifestyle styles for internal automation tools unless the task is explicitly a public marketing surface.
- Do not pick dark-only styles for accessibility-sensitive forms without checking contrast and focus states.
